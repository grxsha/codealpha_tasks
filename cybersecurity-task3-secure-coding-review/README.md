# Secure Coding Review

## Objective

The objective of this project is to perform a security review of a Python application, identify common security vulnerabilities, and recommend secure coding practices to mitigate them.

---

## Tools Used

- Python 3.x
- Manual Code Review
- Bandit (Static Analysis Tool)

---

## Files

- vulnerable_app.py - Contains intentionally insecure code.
- secure_app.py - Improved version with security fixes.
- security_review.md - Documents vulnerabilities and recommendations.

---

## Vulnerabilities Identified

- SQL Injection
- Hardcoded Credentials
- Weak Password Storage
- Poor Exception Handling

---

## Recommendations

- Use parameterized SQL queries.
- Store passwords using hashing.
- Remove hardcoded credentials.
- Validate user input.
- Handle exceptions securely.

---

## Learning Outcomes

- Understand secure coding principles.
- Identify common software vulnerabilities.
- Apply secure coding best practices.
- Perform basic security audits.
